# 公園マップ ピン色・キャッチボール可否の判定基準

最終更新: 2026-07-16（戸田市公式情報の反映に合わせて制定）

## 基本原則

**情報源の信頼度が高い順に優先する。** 上位の情報がある公園は、下位の情報（投票など）で色が変わらない。

| 優先 | 情報源 | 説明 |
|---|---|---|
| 1 | **自治体の公式情報** | 市町村から提供された一覧・回答（例: 戸田市 令和8年7月）。公認バッジ（青✓）付き |
| 2 | **事務局の手動登録・現地調査** | PARKFUL調査230件など、data/parks-data.js に登録した可否 |
| 3 | **ユーザー報告の事務局承認** | 写真・備考付き報告を承認ページで承認したもの |
| 4 | **投票の自動判定** | 👍/👎ワンタップ投票が閾値に達したときの自動反映 |

## ピンの色と、立つ条件

| ピン | 意味 | 立つ条件 |
|---|---|---|
| 🟢 緑 | キャッチボール可 | ①公式情報で可 ②手動登録で可 ③承認済み報告の集計で「できる」≧「できない」 |
| 🔴 赤 | 不可・条件付き | ①公式情報で不可 ②手動登録で不可 ③承認済み報告の集計で「できない」＞「できる」 |
| 🟠 オレンジ | 判定不明（報告あり） | 承認済みだが可否の決定的な報告がない（不明のみ等） |
| 🟡 琥珀（破線） | 確認中 | 未承認の報告・投票がある（下記の自動公開ラインに未達） |
| ⚪ グレー | 情報未登録 | OSM由来の未調査公園。報告を募集中 |
| 🔵 青✓バッジ | 自治体公認 | 上記ピンに重ねて表示。市の公式情報に基づく公園 |

## 投票の自動判定ライン（GAS `_publishVoteState` 実装値）

投票だけで自動的に地図へ反映（緑/赤化）されるのは、次を **すべて** 満たしたとき:

1. **報告合計 3件以上**（できる＋できない＋不明）
2. **できる/できないの決定的回答が1件以上**
3. **多数派が7割以上**（できる vs できない の比率）

- 判定: できる ≧ できない → 可（緑） / できない ＞ できる → 不可（赤）
- 未達の間は「確認中（琥珀）」のまま。事務局が承認ページから手動承認すれば即時反映も可能
- 写真・備考付きの報告は自動公開しない（不適切コンテンツ防止のため必ず事務局承認）

## 優先順位の実装上の担保

- 手動登録（curated）の可否は GAS 集計で上書きされない（`showParkModal` のマージは catchball が null のときだけ GAS 値を採用）
- 自治体公式（official付き）の公園は、投票が反対方向に集まっても**色を変えない**。
  実態と公式情報が食い違う報告が続く場合は、事務局が自治体へ照会して公式情報側を更新する
- 公式情報による「不可」は notes に出典を明記する（例: 芦原ちびっ子広場）

## 運用ルール

- 承認済み公園への新着報告は承認ページに「更新報告」として表示 → 「承認して更新」で票数・写真を反映
- 明らかないたずら・重複は投稿単位で却下（非破壊・却下ログに写真付きで記録）
- 公式情報を受領したら `official:{ city, asOf, name?, hours?, note? }` を付与し、
  「最近の更新」のお知らせ（`parkNews`）にも掲載する
