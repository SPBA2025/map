# 地域コンテンツ（イベント・チーム案内・広告）運用ガイド

最終更新: 2026-07-17

公園マップに、見ている地域（市町村）に連動して「イベント情報」「体験・スクール案内」「広告」を表示する仕組み。
掲載内容は事務局がスプレッドシートで管理し、GAS（`action=local`）が配信する。コード変更・git push は不要。

## 表示される場所

1. **公園の詳細モーダル下部**「この地域の情報（○○市）」— その公園の市町村に該当する最大3件
2. **サイドバー**「この地域の情報」— 地図で見ている市町村に連動して最大3件
3. **左下カード**「○○市エリアの情報」— 市町村に入った時に最大2件をポップアップ表示
   - うるさくならない制御: 同一市町村は1セッション1回まで／1セッション合計2回まで／×で閉じたら新しい掲載が出るまで再表示しない

いずれも該当0件なら何も表示されない。地図が広域表示（ズーム12未満）や県外のときも表示されない。

## スプレッドシート「地域コンテンツ」シートの列

公園マップ用スプレッドシート（フォームの回答があるもの）に「地域コンテンツ」シートを作成する。

| 列 | ヘッダー | 入力 |
|---|---|---|
| A | id | 一意の連番（手入力。計測・重複判定キー） |
| B | 種別 | プルダウン: `イベント` / `チーム・スクール` / `広告` |
| C | タイトル | カードの見出し（例: 所沢市 少年野球体験会） |
| D | 説明 | 60字程度まで |
| E | 対象市町村 | `所沢市,狭山市` のようにカンマ区切り。全県向けは `全県`。表記はマップの市町村選択と同じ正式名 |
| F | 掲載開始日 | 日付。この日から表示 |
| G | 掲載終了日 | 日付。この日を過ぎると自動で消える |
| H | 開催日表示 | 「8/2(日) 9:00〜」など自由記入（任意） |
| I | リンクURL | 詳細・申込ページ（任意） |
| J | 画像URL | Cloudinary にアップロードした画像のURL（任意） |
| K | 提供者名 | **広告のときは必須**（「提供: ○○」と表示される。空だと配信されない） |
| L | 緯度 / M | 経度 | 任意（将来のピン表示用） |
| N | 優先度 | 数値が大きいほど上に表示 |
| O | 掲載 | チェックボックス。OFFにすると即非公開 |
| P | メモ | 事務局用（配信されない） |

- 反映まで**最長10分**（GAS側キャッシュ）。急ぎで消したいときは「掲載」をOFFにして10分待つ
- 広告はカードに **PRバッジ＋「提供: ○○」** が自動表示される（景品表示法のステマ規制対応）

## 掲載ポリシー

1. 掲載対象は**野球・スポーツの振興に資する事業者・団体**に限る（体験会、スクール、大会、用品店等）
2. 掲載前に**事務局が内容を確認**してからシートに入力する
3. 医療・金融・出会い系・射幸性の高いもの・公序良俗に反するものは掲載しない
4. 対価を得た掲載（広告）は種別を「広告」とし、**PR表記と提供者名を必ず表示**する
5. 掲載期間はシートの開始日・終了日で管理する
6. リンク先の内容は掲載者の責任とする

## 効果測定（広告主向けレポート）

GA4 に以下のイベントが送信される（Cookie同意ユーザーのみ）:
- `local_content_impression` — 表示（placement: modal/sidebar/card、city、item_ids、kinds）
- `local_content_click` — クリック（item_id、kind、city、url）

GA4 の探索レポートで item_id 別に 表示回数・クリック数・CTR・市町村別内訳を集計できる。

## 技術メモ

- 配信: `GAS_URL?action=local`（gas/Code.gs `getLocalContent()`、CacheServiceで10分キャッシュ）
- クライアント: `js/parks-map.js` の地域コンテンツセクション（localStorage 60分キャッシュ、`data/local-content.js` が非常用フォールバック）
- 市町村判定: `data/geo-cities.js` の境界ポリゴンでオフライン判定（外部APIを叩かない）
- シート変更後にGASの再デプロイは**不要**（コードを変えたときだけ新バージョンデプロイ）
