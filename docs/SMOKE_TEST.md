# スモークテスト チェックリスト

リファクタリングや大きな変更のデプロイ後に、 必ずこのリストを通すこと。
所要時間: 約10分（PC 7分 + モバイル 3分）

実施環境:
- PC: Chrome（ハードリロード Ctrl+Shift+R で最新を取得してから）
- モバイル: iPhone Safari

---

## チームマップ（/）

- [ ] 初回ロード: スケルトン表示 → 地図が表示される
- [ ] コロプレス色分けが表示される（市町村ごとに色）
- [ ] 市町村クリック → ポップアップが開く（市町村名・統計・登録チーム）
- [ ] ポップアップ排他制御: 別の市町村をクリックすると前のポップアップが閉じる
- [ ] ポップアップの「詳細グラフ」→ モーダルが開く → ESC で閉じる
- [ ] ポップアップの「比較」→ 市町村比較モーダルが開く → 2市町村選択で比較表示
- [ ] ズームイン（11以上）→ チームピンが表示される
- [ ] チームピンクリック → チーム詳細ポップアップ（市町村ポップアップは閉じる）
- [ ] チームリストを開く → 検索・タブ切替・ソートが動く
- [ ] 絞込で0件にする → 空状態カード「絞込をリセット」が表示・動作する
- [ ] サイドバー: カテゴリ・性別・表示スタイル切替が反映される
- [ ] PC: サイドバー折りたたみトグルが動く
- [ ] 「共有」ボタン → URLコピー → 新規タブで開くと絞込状態が復元される
- [ ] チームポップアップ「情報の修正を提案」→ Googleフォームがチーム名入りで開く

## 公園マップ（/parks.html）

- [ ] 公園マーカーが全件表示される（約230件・F12 Network で Places API 呼び出しが**ない**こと）
- [ ] 「キャッチボール可のみ」フィルターが動く
- [ ] 公園一覧: 件数表示・名前順/近い順ソート・クリックでズーム+モーダル
- [ ] 公園マーカークリック → 詳細モーダル → 住所リンクが Google Maps の正しい場所に飛ぶ
- [ ] 「公園を報告」→ 案内バナー表示 → 地図タップ → 座標取得（フォーム or コピー）
- [ ] ESC で報告モード解除・モーダル閉じる
- [ ] 公園名検索が動く
